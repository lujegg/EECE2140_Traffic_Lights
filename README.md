## Basic Functionalities

To achieve the goal of an embedded traffic signal controller system, the following basic functions are required.

| Functionality | Description | Input | Output |
|--------------|------------|-------|--------|
| Traffic Cycle Management | Traffic lights cycle through standard states at set time intervals, alternating between green, yellow, and red. The colors of the lights are all predetermined, and they change repeatedly according to this fixed schedule, just like normal traffic management. | Timer interrupts, predefined time intervals | Updates to light color display |
| Pedestrian Crossing | Allow pedestrians to cross the road safely and temporarily stop vehicle traffic. | Pedestrian button press | Activate the pedestrian signal light and switch the traffic light to red. |
| Emergency Override | A fully red light immediately switches the intersection to a safe state, allowing emergency vehicles to pass. This adds a functionality to allow alternative vehicle types to be entered into our system and protocols. | Emergency sensor trigger | All traffic lights immediately turned red. |
| Power-Loss Contingency | When a system malfunction is detected, it will switch to a fault-safe flashing mode. | Power monitor status | Switch all traffic lights to flashing red mode. |

The purpose of these features is first and foremost to prioritize pedestrian safety, followed by smooth traffic flow and congestion prevention. Secondly, the system is configured to automatically switch to a safety mode in emergency situations to prevent further problems and better protect the entire system.

## Object-Oriented Design (OOP Structure)

### TrafficLight
- lightColor (private, string): What color the light is right now (Red, Yellow, or Green)
- lightType (private, string): What kind of light it is (like a normal light or a left-turn arrow)
- changeColor() (public): Changes the light to the next color
- getColor() (public): Tells us what color to show on the screen

### IntersectionTimer
- currentCycleTime (private, int): Keeps track of how much time has passed for the current light
- emergencyStatus (private, bool): Checks if an emergency is happening right now
- lightLoop() (public): The normal routine: Red → Green → Yellow → Red
- emergencyListener() (public): Watches for emergencies and stops the normal routine if one happens
- powerLowDetector() (public): Notices if power is dropping and makes the lights blink red

### PedestrianSignal
- signalStatus (private, string): Tells people if it's safe to walk or not
- countdownTimer (private, int): How many seconds are left to cross the street
- requestCrossing() (public): Records when someone actually presses the crosswalk button
- changeSignalStatus() (public): Switches the sign between "Walk" and "Don't Walk"

## System Overview - Input, Processing, Output

### Input
The system will receive the following information:
- **Time Parameters:** This is the preset duration for each traffic light color.
- **Pedestrian Requests:** A request is issued when someone presses the pedestrian crossing button; this can be a stop function.
- **Emergency Signals:** Signals triggered when an emergency vehicle approaches; the system will receive these signals.
- **Power Status:** This continuously monitors voltage and power supply.

### Processing
Upon receiving this information, the system will perform the following operations:
- **Check Time Accuracy:** The intersection timer continuously monitors the traffic lights and checks if their duration matches the preset time.
- **Process Pedestrian Buttons:** If someone presses the button, the system will not immediately change the traffic lights but will record the stop signal and then change the traffic lights at an appropriate time.
- **Prioritize Special Cases:** If an emergency signal is detected, such as the arrival of an emergency vehicle, the system will not operate as planned but will forcibly change the traffic lights.
- **Perform traffic light color changes:** The traffic lights perform the required color changes according to the controller's instructions, switching to the desired light color.

### Output
The resulting actions generated by the system:
- **Visual Display:** C++ console output updating at 3-second intervals showing current light arrays (using `-`, `o`, `/` symbols).
- **Pedestrian Indicators:** Updated signals granting or restricting crossing access.
- **System Status:** Logging of emergency overrides or power-loss states for maintenance tracking.
